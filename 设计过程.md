# 设计过程

> 本项目为 LLAgent — 基于 RAG 的小说《圣王》问答 Agent

---

## 讨论内容

### 2026-06-23 — 设计讨论

**话题：系统改进方向**
- 痛点：盲文本匹配对同义/上下位实体不敏感，chunk 边界割裂实体，feedback 缺乏实体知识支撑
- 确定以**层级 metadata 抽取 + 层级检索**为核心改进方向

**话题：实体清洗与实体对齐**
- 从小说中抽取人物/地点/功法/丹药/境界等实体清单，清洗去重
- 实体对齐用于查询增强、检索 boost、feedback 幻觉检测

**话题：对标方向**
- 500 万字不适合纯堆 token，走**"实体锚定"路线** — 离线构建层级索引，线上精确命中
- 定位为 **GraphRAG 轻量变体**：不做全量 KG，只做实体层面的索引增强

**话题：层级索引设计**
- L0 原始 chunk → L1 每章 → L2 每10章 → L3 每100章 → L4 每1000章
- 检索策略：自上而下硬路由 vs 扁平召回后聚合
- Metadata 抽取粒度：每级摘要、主要人物、关键事件

**话题：构造 vs 阅读分离**
- **构造 (离线 Workflow)**：批处理 DAG，确定性构建层级索引
- **阅读 (在线 Agent)**：仅做纯读检索 + 生成，不写数据
- Agent 需要 7 个工具：link_entities / locate_entities / navigate / get_metadata / get_context / search / verify
- 约定索引格式 schema（entity_index.json / hierarchy_tree.json / chunk_index.json）
- 详见 `design.md` 完整设计文档

**话题：RAG Baseline 方案**
- 设计了一个最朴素的 flat RAG 作为层级方案的对照基准
- 文件结构：
  ```
  rag_baseline/
    config.py      — 配置（分块/检索/生成/评估参数）
    indexer.py     — 文本分块（RecursiveCharacterTextSplitter）
    retriever.py   — BM25 + FAISS + RRF 召回
    generator.py   — 简单 LLM 生成（无 feedback 循环）
    pipeline.py    — 完整流水线编排
    evaluate.py    — RAGAS faithfulness 评估
    queries.json   — 10 条测试查询
    run_baseline.py — CLI 入口（answer / eval / interact / index）
  ```
- 比较维度：与层级方案对比 NDCG、faithfulness、token 消耗、幻觉率

**话题：Baseline 跑通**
- 模型改为 Doubao `doubao-seed-1-8-251228`（火山引擎 API）
- "杨奇从城主府偷走了什么？" → 正确答案"伏龙丹"，来源章节 [2, 3, 5]
- 终端 GBK 编码有乱码，不影响逻辑正确
